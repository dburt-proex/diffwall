# DiffWall Demo Media

These versioned assets provide a buyer-facing visual explanation of the current pilot without claiming customer adoption or enterprise readiness.

## Assets

- [`allow-review-halt.svg`](allow-review-halt.svg) — deterministic route and evidence overview.
- [`pilot-workflow.svg`](pilot-workflow.svg) — five-stage AI Coding Governance Pilot sequence.

## Approved uses

- repository documentation;
- private pilot walkthroughs;
- interview or portfolio evidence after the source branch is approved;
- controlled presentations that preserve the maturity disclaimer.

## Prohibited uses

- presenting synthetic graphics as customer screenshots;
- adding customer logos or testimonials without permission;
- claiming production deployment, certification, or measured risk reduction;
- publishing externally before the relevant documentation and publication gates are approved.

## Screenshot and GIF capture checklist

A future live capture should show:

1. an `ALLOW` fixture completing without escalation;
2. a `REVIEW` fixture with findings and suggested owners;
3. a `HALT` fixture publishing evidence before the workflow fails;
4. the policy file and exact DiffWall version or commit;
5. no live secrets, customer data, internal repository names, or unapproved identities;
6. a visible statement that the demonstration is controlled and non-production.

The SVG files are the canonical media source for this increment. Live screenshots and GIFs require a separate capture and publication decision.
